# 회사 리서치 스킬 v2

## 역할
콜드메일 작성 전에 대상 회사와 담당자에 대한 **심층 사전 조사**를 수행합니다.
단순 검색이 아니라, **회사 홈페이지를 직접 방문**하여 1차 정보를 수집하고,
뉴스/기사를 통해 최신 동향을 파악합니다.

핵심: "이 회사가 지금 무엇에 집중하고 있고, 어떤 파트너를 원하는가?"를 파악합니다.

---

## 입력

### 방식 1: CSV 파일 기반 (일괄 리서치)
```
/research data/prospects_sample.csv 리서치해줘
```
CSV에서 company를 읽고, 각 회사별로 심층 조사를 수행합니다.

### 방식 2: 단건 리서치
```
/research サイバーエージェント
/research 메르카리 鈴木裕子 SRE Manager
```

---

## 조사 절차 (회사당 5단계)

### Step 1: 회사 홈페이지 직접 방문

**WebFetch로 회사 공식 사이트를 직접 열람합니다.**

방문할 페이지 (순서대로):

| 페이지 | URL 패턴 예시 | 수집 내용 |
|---|---|---|
| 메인 페이지 | `https://www.{company}.co.jp/` | 회사 미션, 메인 메시지, 핵심 사업 |
| 사업/서비스 페이지 | `/business/`, `/services/`, `/products/` | 주요 사업 라인, 제품/서비스 목록 |
| IR/투자자 페이지 | `/ir/`, `/investors/` | 매출, 중기경영계획, 주력 성장 영역 |
| 뉴스/프레스릴리즈 | `/news/`, `/press/` | 최근 6개월 주요 발표 |
| 채용 페이지 | `/careers/`, `/recruit/` | 어떤 직무를 뽑고 있는지 → 어디에 투자하는지 |
| 파트너십/제휴 페이지 | `/partners/`, `/alliance/` | 어떤 유형의 파트너를 찾는지 |

**URL을 모를 때:** WebSearch로 `{회사명} 公式サイト` 또는 `{회사명} official site`를 먼저 검색하여 URL을 확보한 후 WebFetch합니다.

**수집 포인트:**
- 회사가 스스로 강조하는 키워드/미션 (그대로 콜드메일에 인용 가능)
- 주력 사업 파이프라인 (현재 매출의 핵심 vs 미래 투자 영역)
- 파트너십 기준 (어떤 조건의 파트너를 원하는지)

---

### Step 2: 최근 뉴스/기사 조사

**WebSearch로 최근 6개월~1년 이내의 기사를 검색합니다.**

검색 쿼리:
```
"{회사명}" 2025 OR 2026 (제휴 OR 협업 OR 投資 OR 出資 OR 新事業 OR DX OR AI)
"{회사명}" partnership OR investment OR launch 2025 OR 2026
```

수집 내용:
- 최근 투자/자금 조달
- 신규 사업/서비스 출시
- 제휴/파트너십 발표
- 경영진 교체/조직 개편
- 업계 수상/인증

**중요:** 기사 URL과 날짜를 함께 기록합니다.

---

### Step 3: 관심 분야/투자 방향 분석

Step 1~2에서 수집한 정보를 종합하여 분석합니다:

| 분석 항목 | 판단 기준 |
|---|---|
| **현재 핵심 사업** | 홈페이지에서 가장 크게 다루는 사업 |
| **성장 투자 영역** | IR자료의 중기계획, 채용이 집중된 영역 |
| **기술적 과제** | 채용 공고의 직무 설명, 기술 블로그 |
| **원하는 파트너 유형** | 파트너십 페이지, 최근 제휴 사례의 패턴 |
| **의사결정 문화** | 대기업(=느린 합의) vs 스타트업(=빠른 실행) |

---

### Step 4: 담당자 조사 (contact_name이 있을 경우)

WebSearch로 담당자의 공개 활동을 조사합니다:

```
"{담당자명}" "{회사명}" (登壇 OR 講演 OR インタビュー OR 寄稿 OR ブログ)
"{담당자명}" "{회사명}" LinkedIn OR conference OR keynote
```

수집 내용:
- 최근 발표/강연 주제 (무엇에 관심이 있는지)
- 기고/인터뷰 내용 (어떤 관점을 가지고 있는지)
- 기술 블로그 게시글 (어떤 기술에 집중하는지)
- 경력 이동 이력 (이전 회사 → 현재 회사로 온 이유)

**주의:** 개인 SNS, 사적 정보는 수집하지 않습니다.

---

### Step 5: 콜드메일 연결 포인트 도출

위 조사를 바탕으로, 콜드메일에 직접 활용할 수 있는 포인트를 정리합니다:

- **Hook 후보 3개**: 메일 오프닝에 쓸 수 있는 구체적 문장
- **Pain Point 추정**: 회사 상황에서 추론되는 과제
- **우리 솔루션 연결점**: 어떤 각도로 접근하면 효과적인지
- **피해야 할 점**: 이 회사에 접근할 때 주의사항 (경쟁사 언급 금지 등)

### CSV 일괄 리서치
- 각 회사별로 위 형식의 리서치 보고서를 출력
- 리서치 결과를 반영한 새 CSV를 `data/prospects_researched_YYYYMMDD.csv`로 저장
- 기존 CSV 데이터는 보존하고, 빈 컬럼을 채우거나 기존 데이터를 보강

---

## 리서치 품질 기준

- **홈페이지 직접 방문 필수**: WebSearch만으로 끝내지 않고 반드시 WebFetch로 홈페이지를 읽는다
- 확인 불가능한 항목은 빈칸으로 남김 (추측하지 않음, 단 "~로 추정" 표기는 허용)
- 정보의 시점을 반드시 표기 (2025년 3월 기사 기반 등)
- 동명이인 주의: 회사명과 함께 검색하여 교차 확인
- 개인정보 보호: 개인 연락처, 사적 SNS 등은 수집하지 않음
- **1개 회사당 최소 WebFetch 3회 이상** (홈페이지 메인 + 사업 페이지 + 뉴스 페이지)

---

## 실행 절차

1. 입력 확인 (CSV 경로 또는 단건 정보)
2. CSV인 경우 파일을 Read 도구로 읽기
3. 각 회사별로:
   a. WebSearch로 공식 홈페이지 URL 확보
   b. WebFetch로 홈페이지 주요 페이지 방문 (Step 1)
   c. WebSearch로 최근 뉴스/기사 검색 (Step 2)
   d. 수집 정보 분석 → 관심 분야/투자 방향 도출 (Step 3)
   e. 담당자 정보 검색 (Step 4, contact_name 있을 경우)
   f. 콜드메일 연결 포인트 정리 (Step 5)
4. 리서치 보고서 출력
5. CSV 일괄인 경우 보강된 CSV 저장
6. 이후 `/coldmail`로 바로 연결 가능하도록 안내
