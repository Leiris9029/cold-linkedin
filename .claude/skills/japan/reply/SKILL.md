# 비즈니스 메일 답장 스킬

## 공통 문법 규칙

**일본어 메일 작성 또는 일본 회사에 보낼 때** 반드시 `.claude/skills/japan/_common/GRAMMAR_JA.md` 파일을 먼저 읽고 적용합니다.

---

## 역할
상대가 보낸 메일에 대한 최적의 답장을 자연스럽고 실무적으로 작성합니다.
일본 현업 비즈니스 메일(敬語/丁寧語) 작성에 특화되어 있으며, 템플릿 티가 나지 않는 자연스러운 답장을 생성합니다.

핵심 원칙:
- 상대가 제기한 포인트를 **빠짐없이** 받아준다
- 과잉겸양은 피한다
- 문장 길이는 읽기 쉽게 적당히 끊는다 (2~3문장마다 줄바꿈)
- 상대가 바로 답하기 쉽게 **다음 액션**을 명확히 제시한다

---

## 입력

사용자가 다음 4가지를 제공합니다:

### 1) 상대 메일 원문
```
<<<
(받은 메일 전문)
>>>
```

### 2) 답장에 담고 싶은 요지/의도 (한국어 가능)
- 예: 답변 감사 / Zoom도 가능 / 일정 맞으면 직접 방문도 가능 / PDF 보내겠다 등
- 필요한 추가 메시지/조건

### 3) 제약/선호 (없으면 "없음")
| 항목 | 기본값 | 예시 |
|---|---|---|
| 언어 | 일본어 | 일본어 / 영어 / 혼합 |
| 톤 | 정중 | 매우 정중 / 정중 / 캐주얼 |
| 길이 | 보통 | 짧게 / 보통 / 길게 |
| 시간대 표기 | JST | JST / KST 등 |
| 미팅 옵션 | - | Web(ZOOM) 우선, 대면도 가능 등 |
| 확정 금지 사항 | - | 방문 확정, 계약 확정 등 |
| 첨부/링크 | - | PDF 첨부할 예정, 자료 링크 포함 등 |

### 4) 서명
- 별도 입력 불필요 — 기존에 사용 중인 서명을 그대로 사용한다.
- 사용자가 서명을 명시적으로 제공한 경우에만 그것을 사용한다.

---

## 작업 절차

### A. 포인트 분석
상대 메일에서 핵심 포인트(요청, 우려, 조건, 질문)를 3~6개로 정리한다.

### B. 빠짐없는 응답
그 포인트에 **하나도 빠짐없이** 답한다.

### C. 요지 반영
사용자가 전달하려는 요지를 자연스럽게 포함한다.

### D. 다음 단계 제시
상대가 바로 답하기 쉽도록, 다음 단계를 구체적으로 제시한다:
- 일정 조율 필요 시: 후보시간 2~3개 요청 또는 제시 (입력 요지에 따라 적절히 선택)
- 미팅 시간(예: 30분)과 방식(Zoom/対面)을 명시
- 선택지를 주어 부담을 낮춘다

### E. 톤 제어
- 불필요한 공격적/강요 표현은 금지
- 선택지를 주고 부담을 낮춘다
- 사실관계가 불명확하면 단정하지 말고 「認識違いでしたらご容赦ください」같은 완충을 넣는다

## 일본어 경어 가이드라인

### 기본 경어 패턴
| 상황 | 표현 |
|---|---|
| 메일 수신 감사 | ご連絡いただきありがとうございます |
| 검토 감사 | ご検討いただき誠にありがとうございます |
| 일정 제안 | ご都合のよろしい日時をお知らせいただけますと幸いです |
| 자료 송부 | 資料を添付いたしましたのでご確認ください |
| 방문 제안 | お伺いすることも可能でございます |
| 조건 수락 | 承知いたしました |
| 요청 | お手数ですが〜いただけますと幸いです |
| 미팅 후보 | 以下の日程でご都合はいかがでしょうか |

### 피해야 할 표현
- 「させていただく」남용 (1통에 2회 이내)
- 「〜の方」불필요한 사용
- 이중 경어: 「お伺いさせていただく」→「お伺いする」
- 불필요한 「よろしくお願いいたします」반복 (마지막에 1회)

### 자연스러운 단락 구성
```
[인사/감사 1문장]

[상대 포인트에 대한 응답 - 포인트별 2~3문장씩]

[다음 액션 제시]

[마무리 인사]

[서명]
```

---

## 출력 형식 (반드시 준수)

```
件名：(짧고 자연스러운 제목 1개)

---

(완성된 답장 본문 — 경칭, 단락, 개행, 서명 포함. 바로 복사해서 보낼 수 있는 상태)
```

- 제목은 **1개만** 출력한다 (후보 여러 개 X)
- 추신(P.S.)은 넣지 않는다
- 서명은 기존 서명을 그대로 붙인다
- 분석 과정은 출력하지 않고, 완성 결과만 출력한다

---

## 품질 체크

- [ ] 상대 메일의 모든 포인트에 빠짐없이 답했는가?
- [ ] 사용자의 요지/의도가 모두 반영되었는가?
- [ ] 「させていただく」가 2회 이하인가?
- [ ] 이중 경어가 없는가?
- [ ] 다음 액션이 구체적으로 제시되었는가?
- [ ] 톤이 제약/선호에 맞는가?
- [ ] 불확실한 사실을 단정하지 않았는가?

---

## 실행 절차

1. 사용자 입력 확인 (상대 메일, 요지, 제약, 서명)
2. 상대 메일에서 핵심 포인트 3~6개 추출
3. 각 포인트에 대한 응답 + 사용자 요지 통합 설계
4. 답장 작성 (경어/톤/길이 준수)
5. 품질 체크 통과 확인
6. 출력 형식에 맞춰 결과 출력
