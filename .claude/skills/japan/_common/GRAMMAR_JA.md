## 2. 「の」 삽입 규칙

### 2.1 회사명 + 직함
본문에서 회사명과 직함 사이에 반드시 「の」를 넣습니다.

| NG | OK |
|---|---|
| `RISORIUS Inc. 共同創業者` | `RISORIUS Inc.の共同創業者` |
| `塩野義製薬 BD部長` | `塩野義製薬のBD部長` |

**예외:** 署名(서명) 블록에서는 「の」 생략 가능 (명함 스타일)

### 2.2 명사 + ご相談/ご紹介/ご提案
제목이나 본문에서 명사 뒤에 「の」를 넣습니다.

| NG | OK |
|---|---|
| `効率化ご相談` | `効率化のご相談` |
| `協業ご提案` | `協業のご提案` |
| `反応性予測AIご紹介` | `反応性予測AIのご紹介` |

## 3. 한국 제약사 협업 표현

### 3.1 개발 주체 명확히
우리가 직접 개발 + 한국 제약사와 협업은 **별도 문장**으로 분리합니다.

| NG | OK |
|---|---|
| `韓国の製薬企業と協業し開発しております` | `開発しております。現在、韓国の製薬企業とも...協業を進めております` |

### 3.2 「とも」 사용
"한국에서도 하고 있다"는 뉘앙스를 주기 위해 「とも」를 사용합니다.

```
現在、韓国の製薬企業ともCNS領域で協業を進めております。
```

## 4. CTA (Call-to-Action) 규칙

### 4.1 기본값: 관련 부서/담당자 연결 요청

```
ご関心をお持ちいただけましたら、関連部署・ご担当者様へおつなぎいただけますと幸いです。
```

### 4.2 금지 사항
사용자가 명시적으로 지정하지 않은 CTA는 추가하지 않습니다.

- ❌ 「ウェブ面談」「Zoom」「15〜20分」 등 미팅/시간 언급
- ❌ 「資料をお送りします」 등 자료 송부 언급
- ❌ 「デモをお見せします」 등 데모 제안

### 4.3 문장 연결
CTA 앞에 「ご関心をお持ちいただけましたら」를 붙여 자연스럽게 연결합니다.

## 5. 敬語 규칙

### 5.1 依頼トーン (요청 톤 완화)

| NG (명령조) | OK (겸양) |
|---|---|
| `お知らせください。` | `お知らせいただけますと幸いです。` |
| `ご教示ください。` | `ご教示いただけますと幸いです。` |
| `ご連絡ください。` | `ご連絡いただけますと幸いです。` |

### 5.2 させていただく 제한
1통당 **최대 1회**만 사용합니다.

### 5.3 会社呼称 (회사 호칭)

- **첫 1회만**: 정식 통칭 + 様 (예: `塩野義製薬様`)
- **두 번째 이후**: 반드시 `貴社`로 대체
- `御社`는 사용하지 않음 (문어체에서는 `貴社`)

## 6. 단정 완화

회사/상대방에 대한 사실을 언급할 때 단정하지 않습니다.

| NG | OK |
|---|---|
| `〜です` (단정) | `〜と存じます` / `〜と拝察いたします` |
| `〜をリードされている` | `〜に携わっていらっしゃる` / `〜をご担当` |
| `〜を統括されている` | `〜に携わっていらっしゃる` |

## 7. 검증되지 않은 정보 금지

### 7.1 구체적 수치
검증되지 않은 수치/통계는 사용 금지

| NG | OK |
|---|---|
| `50%削減` (근거 없음) | `負荷軽減に寄与いたします` |
| `大幅に効率化` | `業務効率化を支援いたします` |

### 7.2 상대 회사 제품명
틀릴 위험이 있으므로 구체적 제품명 직접 언급 피함

| NG | OK |
|---|---|
| `LEQEMBIの臨床開発において` | `CNS治療薬の開発において` |
| `Pitolisantの承認申請` | `睡眠障害領域` |

## 8. 문장 연결 규칙

### 8.1 앞뒤 문장 논리적 흐름
갑자기 다른 주제로 전환하지 않습니다.

| NG | OK |
|---|---|
| `...ご連絡いたしました。非機密情報のみを取り扱い...` | `...ご連絡いたしました。現在、韓国の製薬企業とも...` |

→ "연락드렸습니다" 다음에 갑자기 "비기밀 정보만 취급합니다"는 연결이 부자연스러움

### 8.2 불필요한 내용 삭제
문맥상 흐름을 끊는 내용은 과감히 삭제합니다.
