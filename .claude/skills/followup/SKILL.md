# 후속(Follow-up) 메일 작성 스킬 v2

## 역할
콜드메일 발송 후 응답이 없는 대상에게 보내는 후속 메일을 작성합니다.
핵심은 **매번 새로운 가치를 제공**하는 것입니다. 단순 리마인드는 스팸입니다.

원칙: "이전 메일 보셨나요?"는 절대 쓰지 않습니다. 매 후속 메일마다 답장할 새로운 이유를 제공합니다.

---

## 입력

사용자가 다음 정보를 제공합니다:
- **회사명 / 담당자명**: 필수
- **이전 메일 내용 또는 요약**: 어떤 내용으로 첫 메일을 보냈는지
- **후속 단계**: 1차, 2차, 3차 중 몇 번째 후속인지
- **언어**: ko / en / ja
- **추가 컨텍스트**: 새로운 소식, 업데이트, 이벤트 등 (선택)
- **response_status**: 완전 무응답 / 열람함(오픈 추적) / 일부 반응 (선택)

CSV 파일 기반 일괄 생성도 지원합니다. `data/` 폴더의 CSV에 `followup_stage` 컬럼을 추가하면 됩니다.

---

## 단계별 전략 (고도화)

### 1차 후속 (첫 메일 발송 후 3-5일)
- **전략: 새로운 가치 추가**
- 이전 메일을 직접 언급하지 않고, 관련된 새로운 가치를 제공
- 예: 관련 사례 스터디, 업계 데이터, 유용한 인사이트
- 톤: 가볍고 도움을 주는 느낌
- 길이: 2-3문장

**구조:**
```
{호칭},

[새로운 가치 1가지 - 사례, 데이터, 인사이트 등]

[이전 제안과의 자연스러운 연결 1문장]

[가벼운 CTA]
```

**제목 패턴:**
- "{업종} {과제} 관련 데이터를 발견해서"
- "Quick thought on {pain_point}"
- "先日の件に関連して、{업종}の事例を見つけました"

### 2차 후속 (1차 후속 후 5-7일)
- **전략: 다른 각도에서 접근**
- 완전히 다른 가치 포인트로 접근 (1차와 같은 내용 반복 금지)
- 예: 경쟁사 동향, ROI 계산, 다른 부서 사례
- 톤: 약간 더 직접적
- 길이: 3-4문장

**구조:**
```
{호칭},

[다른 각도의 관찰 또는 질문]

[이 각도에서의 가치 제안 + 구체적 숫자]

[이전보다 살짝 더 직접적인 CTA]
```

**제목 패턴:**
- "{경쟁사/유사기업}도 같은 고민을 했었습니다"
- "Different angle on {topic}"
- "{회사}の{과제}について、別の視点から"

### 3차 후속 (2차 후속 후 7-10일)
- **전략: 브레이크업 (마지막 메일)**
- "마지막으로 연락드립니다"의 역심리 활용 (실제로 답장률이 가장 높은 단계)
- 부담 없이, 문을 열어두되 깔끔하게 마무리
- 톤: 담백하고 존중하는
- 길이: 2-3문장

**구조:**
```
{호칭},

[마지막 연락이라는 신호 - 존중하는 톤]

[혹시 타이밍이 아니라면 언제든 연락 가능하다는 메시지]

[문을 열어두는 마무리]
```

**제목 패턴:**
- "마지막으로 여쭤봅니다"
- "Closing the loop"
- "最後のご連絡になります"

## 응답 상태별 톤 조정

`response_status`에 따라 톤을 미세 조정합니다:

| 상태 | 조정 |
|---|---|
| **완전 무응답** | 기본 전략 그대로 |
| **열람함 (오픈 추적)** | 관심은 있으나 바쁜 것으로 판단 → 더 짧고 핵심적으로, CTA 부담 낮춤 |
| **일부 반응** (링크 클릭 등) | 관심 높음 → 더 구체적인 제안, CTA를 명확하게 |

---

## 언어별 톤 상세

### 한국어
- 1차: "~드리고 싶어서 다시 연락드립니다" (부드럽게)
- 2차: "~에 대해 한 가지 더 말씀드리고 싶었습니다" (약간 직접적)
- 3차: "바쁘신 줄 알기에 마지막으로 연락드립니다" (존중)
- **금지**: "이전 메일 확인하셨나요?", "리마인드 드립니다"

### 영어
- 1차: Share value, don't remind. "I came across this and thought of {company}..."
- 2차: Different angle. "One more thing I wanted to share..."
- 3차: Clean break. "I don't want to clog your inbox..."
- **Ban**: "Just following up", "Bumping this", "Per my last email"

### 일본語
- 1차: 「先日の件に関連して、もう一点お伝えしたいことがございまして」
- 2차: 「別の観点から、お役に立てる情報がございます」
- 3차: 「何度もご連絡差し上げ恐縮です。最後のご連絡とさせていただきます」
- **禁止**: 「先日のメールはご覧いただけましたでしょうか」

## 이전 메일과의 연속성

후속 메일은 이전 메일과 **스레드로 묶이는 것을 전제**로 작성합니다:
- 이전 메일 내용을 장황하게 반복하지 않음
- 하지만 맥락은 유지 (같은 주제의 다른 각도)
- 제목: 이전 메일에 "Re:"를 붙이는 것도 유효한 전략 (2차부터)

## 자동 품질 체크

- [ ] 이전 메일과 동일한 내용을 반복하고 있지 않은가?
- [ ] 새로운 가치 또는 각도가 포함되어 있는가?
- [ ] "이전 메일 보셨나요?" 류의 표현이 없는가?
- [ ] 해당 단계의 톤에 맞는가? (1차=부드럽게, 3차=담백하게)
- [ ] 길이 제한을 준수하는가?

---

## 실행 절차

1. 사용자 입력 확인 (대상, 단계, 언어 등)
2. 이전 메일 내용이 있으면 참고하여 맥락 파악
3. 이전 메일과 **다른** 각도/가치를 설계
4. 단계별 전략에 맞는 후속 메일 작성
5. 품질 체크 통과 확인
6. 결과 출력 및 필요 시 파일 저장
