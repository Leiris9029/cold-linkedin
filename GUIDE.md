## 1. 시작하기 전에

### 필요한 것

- **Claude Code** (VS Code 확장 또는 CLI)
- 이 프로젝트 폴더를 작업 디렉토리로 열기

### 최초 설정 (1회만)

```
1. VS Code에서 이 폴더(Cold)를 엽니다
2. Claude Code를 실행합니다
3. 끝. 별도 설치나 설정 없이 바로 사용 가능합니다.
```

스킬은 `.claude/skills/` 폴더에 이미 포함되어 있어 자동으로 인식됩니다.

## 2. 프로젝트 구조

```
Cold/
├── .claude/skills/
│   ├── coldmail/SKILL.md    ← 콜드메일 생성 (설득 프레임워크 + 업종별 차별화)
│   ├── review/SKILL.md     ← 메일 검수 (팩트체크 + 수치 감사 + 문법)
│   ├── linkedin/SKILL.md   ← LinkedIn DM 변환 (연결 요청·DM·팔로업) [NEW]
│   ├── followup/SKILL.md   ← 후속 메일 생성 (단계별 전략)
│   ├── research/SKILL.md   ← 회사/담당자 사전 리서치
│   └── abtest/SKILL.md     ← A/B 테스트 변형 생성
├── data/
│   └── prospects_sample.csv ← 영업 대상 CSV (고도화 컬럼 포함)
├── output/                  ← 생성된 메일이 저장되는 폴더
└── GUIDE.md                 ← 이 문서
```

| 폴더 | 용도 | 누가 편집하나 |
|---|---|---|
| `.claude/skills/` | 스킬 정의 (수정할 일 거의 없음) | 관리자 |
| `data/` | 영업 대상 리스트 CSV | **영업팀 전원** |
| `output/` | 생성된 메일 결과물 | 자동 생성 |

## 3. 전체 워크플로우

```
┌─────────────────────────────────────────────────────────┐
│                    SALES MAIL WORKFLOW                    │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Step 1. CSV에 타겟 리스트 작성 (기본 정보만 OK)         │
│     ↓                                                    │
│  Step 2. /research 로 회사/담당자 리서치 자동화           │
│     ↓    → 고도화 컬럼 자동 채움                         │
│     ↓    → data/prospects_researched_날짜.csv 생성       │
│     ↓                                                    │
│  Step 3. /coldmail 로 맞춤 메일 생성                     │
│     ↓    → 설득 프레임워크 자동 선택 (PAS/AIDA/BAB)      │
│     ↓    → 업종별 톤/용어 자동 적용                      │
│     ↓    → 직함 기반 CTA 자동 결정                       │
│     ↓                                                    │
│  Step 4. /review 로 메일 검수                             │
│     ↓    → 팩트체크 (Hallucination 검출)                 │

│     ↓    → 문맥/문법 검수 (자연스러움 확인)              │
│     ↓    → 문제 발견 시 자동 수정                        │
│     ↓                                                    │
│  Step 4.5. /linkedin 으로 LinkedIn DM 변환 (선택) ★ NEW  │
│     ↓    → 콜드메일 → 연결 요청 노트 3종 (A/B/C)        │
│     ↓    → 콜드메일 → 첫 DM 2종 (A/B)                   │
│     ↓    → 팔로업 DM 1종                                 │
│     ↓                                                    │
│  Step 5. /abtest 로 A/B 변형 생성 (선택)                 │
│     ↓    → 제목/Hook/CTA 변형 테스트                     │
│     ↓                                                    │
│  Step 6. 메일 발송                                       │
│     ↓                                                    │
│  Step 7. /followup 1차 (3~5일 후, 새로운 가치 제공)      │
│     ↓                                                    │
│  Step 8. /followup 2차 (5~7일 후, 다른 각도)             │
│     ↓                                                    │
│  Step 9. /followup 3차 (7~10일 후, 브레이크업)           │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 4. CSV 파일 작성법

### Step 1: `data/` 폴더에 CSV 파일을 만듭니다

파일명은 자유입니다. 예시:
- `prospects_japan_q1.csv` (일본 1분기 타겟)
- `prospects_korea.csv` (한국 타겟)
- `enterprise_leads.csv` (엔터프라이즈 리드)

### Step 2: 컬럼 형식

CSV는 **3단계 레벨**로 나뉩니다. 처음에는 기본만 채우고, `/research`가 나머지를 채워줍니다.

#### Level 1: 필수 컬럼 (최소 이것만 있으면 동작)

| 컬럼명 | 설명 | 예시 |
|---|---|---|
| `company` | 대상 회사명 | `楽天グループ` |
| `contact_name` | 담당자 이름 | `佐藤花子` |
| `language` | 메일 작성 언어 | `ko`, `en`, `ja` |

#### Level 2: 기본 컬럼 (직접 채우기 권장)

| 컬럼명 | 설명 | 예시 |
|---|---|---|
| `contact_title` | 담당자 직함 | `CTO`, `VP of Eng` |
| `email` | 이메일 주소 | `sato@rakuten.co.jp` |
| `industry` | 업종 | `Eコマース`, `핀테크` |
| `pain_point` | 핵심 과제/고민 | `月200時間のセキュリティ監査対応` |
| `our_solution` | 제안할 가치 | `監査対応工数を80%削減` |

#### Level 3: 고도화 컬럼 (/research가 자동으로 채워줌)

| 컬럼명 | 설명 | 효과 |
|---|---|---|
| `company_size` | 직원수/매출 규모 | 회사 규모에 맞는 톤 조정 |
| `competitor_used` | 현재 사용 중인 경쟁 솔루션 | 경쟁사 대비 차별점 강조 |
| `trigger_event` | 최근 트리거 이벤트 | AIDA 프레임워크 자동 선택 |
| `mutual_connection` | 공통 지인/접점 | Referral 프레임워크 자동 선택 |
| `contact_linkedin_summary` | 담당자 관심사 | Hook 개인화 강화 |
| `tone_override` | 톤 오버라이드 | `formal`/`casual`/`bold` |
| `cta_type` | CTA 유형 지정 | `meeting`/`demo`/`resource`/`intro` |
| `notes` | 추가 참고사항 | 자유 텍스트 |

### CSV 작성 팁

> **pain_point**와 **our_solution**에 숫자를 넣으면 메일 퀄리티가 크게 올라갑니다.

**좋은 예:**
```
pain_point: "멀티클라우드 환경에서 월 비용 추적이 어렵고 예산 초과 빈번"
our_solution: "통합 모니터링으로 비용 30% 절감 + 실시간 예산 알림"
```

**나쁜 예:**
```
pain_point: "IT 문제가 있음"
our_solution: "우리 제품이 좋음"
```

### 언어 코드

| 코드 | 언어 | 기본 톤 |
|---|---|---|
| `ko` | 한국어 | 비즈니스 존댓말 (~합니다 체) |
| `en` | English | Professional but conversational |
| `ja` | 日本語 | ビジネス敬語 (丁寧語) |

## 5. 회사 리서치 (/research) **NEW**

콜드메일 작성 전 대상 회사/담당자를 웹에서 자동 조사합니다.
CSV의 고도화 컬럼(Level 3)을 자동으로 채워줍니다.

### 사용법

**CSV 일괄 리서치:**
```
/research data/prospects_sample.csv 의 회사들을 리서치해줘
```

**단건 리서치:**
```
/research 카카오엔터프라이즈 이지현 DevOps Lead
```

### 어떤 정보를 수집하나?

| 수집 항목 | 방법 | 활용 |
|---|---|---|
| 회사 규모 | 웹 검색 | 톤 조정 |
| 최근 뉴스/이벤트 | 뉴스 검색 | 메일 Hook (트리거) |
| 기술 스택 | 블로그/채용공고 | 경쟁사 비교 포인트 |
| 담당자 관심사 | LinkedIn 검색 | 개인화된 오프닝 |
| 채용 동향 | 채용공고 검색 | 페인포인트 힌트 |

### 출력 결과

- 화면에 리서치 요약 출력
- CSV 일괄: `data/prospects_researched_YYYYMMDD.csv` 파일로 보강된 CSV 저장
- 각 회사별 **콜드메일 활용 포인트** (추천 Hook, Framework, CTA) 제안

### 추천 워크플로우

```
1. CSV에 Level 1~2 컬럼만 채움 (회사명, 담당자, 업종 정도)
2. /research 실행 → Level 3 자동 채움
3. 결과 CSV 확인 → 필요시 수동 보정
4. /coldmail 실행 → 훨씬 개인화된 메일 생성
```

## 6. 콜드메일 생성 (/coldmail)

### 방법 A: CSV 파일로 일괄 생성

```
/coldmail data/prospects_sample.csv 파일을 읽고 각 회사별로 콜드메일을 만들어줘
```

### 방법 B: 1건씩 직접 입력

```
/coldmail
회사: 라인
담당자: 야마다 타로 (CTO)
업종: 메신저 플랫폼
페인포인트: 글로벌 서비스 확장 시 인프라 관리 복잡성
제안: 멀티리전 자동 배포 및 모니터링 솔루션
언어: ja
```

### 방법 C: 간단하게

```
/coldmail 네이버클라우드 김철수 VP에게 한국어로 콜드메일 써줘. 우리 솔루션은 클라우드 비용 최적화야.
```

### v2에서 달라진 점

| 기능 | v1 | v2 |
|---|---|---|
| 설득 구조 | 고정 템플릿 | **4가지 프레임워크 자동 선택** (PAS/AIDA/BAB/Referral) |
| 제목 | 범용적 | **5가지 공식** 중 상황 맞춤 선택 |
| CTA | 미팅 제안 고정 | **직함 기반 자동 결정** (meeting/demo/resource/intro) |
| 업종 차별화 | 없음 | **업종별 용어/톤/가치 각도 자동 조정** |
| 톤 | 언어별 1가지 | **4단계** (기본/formal/casual/bold) |
| 품질 체크 | 없음 | **7항목 자동 검증**, 미통과 시 재작성 |

### 프레임워크 자동 선택 기준

| 데이터 상황 | 선택되는 프레임워크 |
|---|---|
| `pain_point`가 구체적 | **PAS** (Problem → Agitate → Solution) |
| `trigger_event`가 있음 | **AIDA** (Attention → Interest → Desire → Action) |
| 유사 사례/레퍼런스 보유 | **BAB** (Before → After → Bridge) |
| `mutual_connection`이 있음 | **Referral** (소개 기반) |

직접 지정도 가능:
```
/coldmail ... 프레임워크는 AIDA로 써줘
```

## 7. 메일 검수 (/review) **NEW**

`/coldmail`로 생성된 메일의 **사실 정확성, 수치 신뢰성, 문법 자연스러움**을 자동 검수합니다.
AI가 생성한 메일에는 Hallucination(날조)이 포함될 수 있으므로, 발송 전 반드시 검수를 권장합니다.

### 사용법

**CSV 전체 검수:**
```
/review output/coldmails_mailmerge.csv
```

**MD 파일 검수:**
```
/review output/coldmails_20260202.md
```

**특정 회사만 검수:**
```
/review output/coldmails_mailmerge.csv, Takeda만 검수해줘
```

**검수 + 자동 수정:**
```
/review output/coldmails_mailmerge.csv, 수정까지 해줘
```

### 검수 3단계

| 단계 | 내용 | 검출 예시 |
|---|---|---|
| **1단계: 팩트체크** | 회사 고유 사실 웹 검증 | `China顆粒` (날조 제품명), `買収検討` → `買収合意` |
| **2단계: 수치 감사** | 모든 %, $, 億円 출처 확인 | `60% 삭감` (our_products.md에 없음 → 날조) |
| **3단계: 문맥/문법** | 수정 후 깨진 표현 탐지 | `by significantly` (문법 파괴), `最大大幅に` (중복) |

### 심각도 분류

| 심각도 | 대응 | 예시 |
|---|---|---|
| CRITICAL | 즉시 수정/삭제 | 날조 제품명, 존재하지 않는 수치 |
| WARNING | 사용자 확인 후 수정 | 부정확한 딜 상태, 오래된 정보 |
| INFO | 선택적 수정 | 보수적 표현 vs 실제 차이 |

### 검수 리포트

검수 완료 후 `output/review_YYYYMMDD.md`에 상세 리포트가 저장됩니다.
리포트에는 발견된 모든 문제, 심각도, 수정안이 포함됩니다.

### 추천 워크플로우

```
1. /coldmail 로 메일 생성
2. /review 로 전체 검수 실행
3. 검수 리포트 확인 → 수정 승인
4. 자동 수정 적용 → 재검수 (2차 오류 확인)
5. /abtest 또는 메일 발송
```

## 8. LinkedIn DM 변환 (/linkedin) **NEW**

콜드메일을 **LinkedIn 연결 요청 노트**, **첫 DM**, **팔로업 DM**으로 변환합니다.
이메일과 LinkedIn을 병행하면 접촉 채널이 늘어 응답률이 올라갑니다.

### 사용법

**콜드메일 텍스트를 붙여넣어서 변환:**
```
/linkedin
아래 콜드메일을 LinkedIn DM으로 변환해줘:
[메일 본문 붙여넣기]
```

**CSV 특정 행 지정:**
```
/linkedin output/260203.csv Row 5 변환해줘
```

**직접 입력:**
```
/linkedin
회사: Takeda
담당자: 田中太郎 (VP of Data Science)
언어: ja
제안: AI 기반 임상시험 데이터 분석 자동화
```

### 출력 구성

| 항목 | 개수 | 글자 수 | 설명 |
|---|---|---|---|
| **연결 요청 노트** | 3종 (A/B/C) | 200자 이내 | LinkedIn 연결 시 첨부하는 짧은 메시지 |
| **첫 DM** | 2종 (A/B) | 600~900자 | 연결 수락 후 보내는 본격 제안 메시지 |
| **팔로업 DM** | 1종 | 300~500자 | 첫 DM 무응답 시 5~7일 후 발송 |

### 변형 전략

**연결 요청 노트 (200자 이내):**

| 변형 | 전략 | 예시 |
|---|---|---|
| A | 인물 기반 | "○○님의 △△ 발표를 인상 깊게 봤습니다" |
| B | 회사 기반 | "御社の△△プロジェクトに関心があり" |
| C | 주제 기반 | "AI×創薬分野で意見交換させていただきたく" |

**첫 DM (600~900자):**

| 변형 | 톤 | 적합한 대상 |
|---|---|---|
| A | 격식체 | 대기업 임원, 보수적 업종 |
| B | 부드러운 톤 | 스타트업, 테크 기업 |

### 절대 규칙

- 콜드메일에 없는 새로운 수치/통계를 **절대 추가하지 않음**
- `data/our_products.md`에 없는 제품 사양을 **절대 날조하지 않음**
- 사용자가 지시하지 않은 CTA를 **절대 임의 추가하지 않음**
- 연결 요청 노트는 **반드시 200자 이내** (LinkedIn 제한)

### 추천 워크플로우

```
1. /coldmail 로 메일 생성 → /review 검수 완료
2. /linkedin 으로 동일 대상 LinkedIn DM 변환
3. 이메일 발송 + LinkedIn 연결 요청 동시 진행
4. 연결 수락 시 → 첫 DM 발송
5. 무응답 시 → 팔로업 DM 발송 (5~7일 후)
```

## 9. A/B 테스트 (/abtest)

콜드메일의 제목, 오프닝, CTA 등을 변형하여 테스트용 버전을 생성합니다.

### 사용법

```
/abtest data/prospects_sample.csv 첫 번째 회사 메일로 제목 A/B 테스트 만들어줘
```

또는 이미 생성된 메일에 대해:
```
/abtest 위에서 만든 田中씨 메일로 CTA A/B 테스트 해줘
```

### 테스트 가능 요소

| 요소 | 영향 | 예시 |
|---|---|---|
| **제목** (기본) | 오픈율 | 질문형 vs 숫자형 vs 가치직행형 |
| **Hook** | 읽기 시작 여부 | 트리거 이벤트 vs 페인포인트 직행 |
| **CTA** | 답장률 | 미팅 제안 vs 자료 공유 vs 질문 |
| **프레임워크** | 전체 설득력 | PAS vs AIDA vs BAB |
| **톤** | 브랜드 인식 | professional vs casual vs bold |

### 출력 예시

```
🧪 A/B Test: SmartWorks Inc / 田中太郎
📌 테스트 요소: 제목 (Subject Line)

🅰️ Variant A (질문형)
  제목: SmartWorksの運用コスト、どう管理されていますか？

🅱️ Variant B (숫자형)
  제목: 同業他社が運用コストを50%削減した方法

📊 테스트 가이드
- 예상: A는 호기심 유발, B는 구체적 가치로 클릭 유도
- 권장: 각 변형 최소 20건 발송
- 측정: 오픈율 비교
```

## 11. 고도화 기능 상세

### 설득 프레임워크

메일이 단순 "정보 전달"이 아닌 "설득"이 되도록 검증된 프레임워크를 적용합니다:

**PAS (Problem → Agitate → Solution)**
```
[Problem] "인프라 스케일링에 수동 작업이 60%를 차지하고 계신다고 들었습니다."
[Agitate] "AI 트래픽이 3배 증가하면 이 비율은 더 올라갈 수밖에 없습니다."
[Solution] "유사 규모 기업에서 자동 스케일링 도입 후 수동 작업을 80% 줄였습니다."
```

**AIDA (Attention → Interest → Desire → Action)**
```
[Attention] "시리즈C 투자 축하드립니다."
[Interest] "글로벌 확장 시 멀티클라우드 비용 관리가 과제가 되곤 합니다."
[Desire] "유사 기업에서 통합 모니터링으로 월 비용 30%를 절감했습니다."
[Action] "15분 통화로 구체적인 방법을 공유드릴 수 있을까요?"
```

### 업종별 차별화

같은 솔루션이라도 업종에 따라 다른 용어와 가치 각도로 작성됩니다:

| 업종 | 용어 예시 | 가치 각도 |
|---|---|---|
| 제조업 | DX, 스마트팩토리, 라인 가동률 | 비용 절감, 운영 효율 |
| 핀테크 | 컴플라이언스, 트랜잭션, 보안 | 규제 대응, 처리 속도 |
| IT/SaaS | DevOps, CI/CD, 클라우드 네이티브 | 개발 생산성, 배포 속도 |
| 이커머스 | 전환율, 옴니채널, 피크 트래픽 | 매출 증대, 고객 이탈 방지 |

### 자동 품질 체크 (7항목)

메일 작성 후 자동으로 검증됩니다:

1. 회사명/담당자명이 정확히 포함되어 있는가?
2. 업종/상황에 맞는 구체적 내용이 있는가?
3. 숫자(%, 금액, 기간)가 최소 1개 포함되어 있는가?
4. 길이 제한을 준수하는가? (한국어 200자, 영어 100단어, 일본어 250자)
5. CTA가 명확하고 예/아니오로 답할 수 있는가?
6. 스팸 트리거 단어가 없는가?
7. **이 메일을 다른 회사에 보내도 되는가?** → YES면 개인화 부족, 재작성

## 13. 잘 쓰는 팁

### 메일 퀄리티 높이는 법

| 팁 | 나쁜 예 | 좋은 예 |
|---|---|---|
| 숫자를 넣어라 | "비용 절감" | "운용 비용 50% 절감" |
| 페인포인트를 구체적으로 | "IT 문제" | "월 200시간의 보안 감사 대응" |
| 트리거를 활용하라 | (없음) | "시리즈C 축하드립니다" |
| 경쟁사를 언급하라 | (없음) | "Jenkins에서 전환한 기업 사례" |

### 톤 오버라이드 활용

CSV의 `tone_override` 또는 직접 지정으로 톤을 바꿀 수 있습니다:

```
/coldmail ... 톤은 casual로 써줘
/coldmail ... bold하게 써줘
```

| 톤 | 적합한 상황 | 예시 (한국어) |
|---|---|---|
| 기본 | 대부분의 상황 | "~에 대해 제안드리고자 연락드립니다" |
| `casual` | 스타트업, 테크 기업 | "~에 대해 한번 얘기 나눠보면 좋을 것 같아요" |
| `formal` | 대기업, 공공기관 | "~에 대해 말씀드릴 기회를 주시면 감사하겠습니다" |
| `bold` | 임팩트 필요할 때 | "솔직히 말씀드리면, 연간 2억을 아낄 수 있습니다" |

### 수정 요청

생성된 메일이 마음에 안 들면 바로 수정 요청 가능합니다:

```
2번째 메일 톤을 좀 더 캐주얼하게 바꿔줘
田中씨 메일에 최근 DX 추진 언급을 추가해줘
전체적으로 CTA를 데모 제안 대신 자료 공유로 바꿔줘
프레임워크를 PAS 대신 AIDA로 바꿔서 다시 써줘
```

## 15. FAQ / 트러블슈팅

### Q: 스킬이 인식되지 않아요
**A:** 이 프로젝트 폴더(Cold)를 VS Code에서 열었는지 확인하세요. `.claude/skills/` 폴더가 있는 디렉토리에서 Claude Code를 실행해야 합니다.

### Q: CSV 파일이 깨져요
**A:** CSV 파일을 UTF-8 인코딩으로 저장하세요. 엑셀에서 저장할 때 "CSV UTF-8 (쉼표로 분리)" 형식을 선택합니다.

### Q: 고도화 컬럼을 다 채워야 하나요?
**A:** 아닙니다. 필수 3개(company, contact_name, language)만 있으면 동작합니다. 나머지는 채울수록 퀄리티가 올라가는 것이고, `/research`가 자동으로 채워주기도 합니다.

### Q: 영어/일본어 메일 톤이 어색해요
**A:** 생성 후 "좀 더 자연스럽게 다듬어줘" 또는 "ネイティブチェックして" 라고 요청하면 수정해줍니다. `tone_override`를 `casual`로 설정하면 더 자연스러운 경우가 많습니다.

### Q: 한 번에 몇 건까지 생성 가능한가요?
**A:** 한 번에 10~20건 정도가 적당합니다. 그 이상이면 파일을 나눠서 실행하세요.

### Q: A/B 테스트 결과는 어떻게 관리하나요?
**A:** `/abtest`는 변형 메일 생성만 담당합니다. 실제 발송 후 오픈율/답장률은 메일 발송 도구(Mailchimp, HubSpot 등)에서 트래킹하세요. 결과를 알려주시면 다음 메일에 반영합니다.

### Q: 다른 팀원도 쓸 수 있나요?
**A:** 이 프로젝트 폴더를 공유하면 (Git, 공유 드라이브 등) 누구나 동일하게 사용 가능합니다. Claude Code만 설치되어 있으면 됩니다.

### Q: 프레임워크를 강제로 지정하고 싶어요
**A:** 자동 선택을 무시하고 직접 지정할 수 있습니다:
```
/coldmail ... 프레임워크는 PAS로 써줘
```

### Q: /review는 반드시 실행해야 하나요?
**A:** 강력히 권장합니다. AI가 생성한 메일에는 날조된 수치나 잘못된 사실이 포함될 수 있습니다. 실제 사례로 "60% 삭감", "87% 정확도" 등 `our_products.md`에 없는 수치가 자동 생성되었던 케이스가 있습니다. `/review`는 이런 문제를 발송 전에 잡아줍니다.

### Q: /linkedin은 콜드메일 없이도 쓸 수 있나요?
**A:** 네, 직접 입력 모드로 회사/담당자/제안 내용을 직접 넣어서 LinkedIn DM을 생성할 수 있습니다. 다만 `/coldmail`로 먼저 메일을 만든 뒤 변환하면 메시지 일관성이 높아집니다.

### Q: 연결 요청 노트가 200자를 초과하면?
**A:** LinkedIn 연결 요청은 300자(영문 기준) 제한이 있습니다. `/linkedin` 스킬은 200자 이내로 생성하며, 초과 시 자동으로 압축합니다.

### Q: /review에서 자동 수정하면 원본이 덮어씌워지나요?
**A:** 네, 대상 파일을 직접 수정합니다. 수정 전에 반드시 사용자 승인을 받으며, 수정 건수와 내용을 상세히 보여준 후 진행합니다. 중요한 파일은 미리 백업해두세요.

### Q: 검수에 시간이 얼마나 걸리나요?
